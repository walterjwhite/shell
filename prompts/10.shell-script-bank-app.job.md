draft a shell script (/bin/sh posix-compliant) banking app using only gnu core utils (awk, grep, sed)

## Data Structure

Account {
id int,
name string
}

Account Balance {
account id int,
transaction id int,
balance double
}

Transaction {
id int,
account id int,
datetime timestamp (stored as a UNIX epoch),
amount double,
description string,
}

## Notes

1. database will be 3 CSV files
2. as this is a 'data' app, repository / work tree will be initialized in $HOME/.data/banking (name of app is banking)
3. this will utilize the lib git/data.app.sh library

## Core functions

1. add transaction (credit / debit)
2. get account balance
3. compute interest for account using average account balance for the month

## Transaction Function

1. add transactions (credit / debit) to a specific account
2. update account balance for the account
3. commit the changes to git, with the commit message '<account id> - <transaction description>'
4. push changes

## cmdline syntax

banking transact <account id> <amount> <description> <transaction datetime, if empty, will use current timestamp>
the transaction will be performed for the given account id, adding an account balance and transaction record
the changes will be committed and pushed

banking interest
will compute the interest for all accounts, then add an interest transaction and update the account balance accordingly
the transaction description will be 'interest - YYYY-mm'
the changes will be committed and pushed

banking balance <account id>
will print the current account balance
